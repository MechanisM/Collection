Одиночные методы

Одиночные методы - это такие методы, которые осуществляют манипулиции с данными, по указанной прямой ссылке. Частично мы уже разбирали некоторые методы, которые могли работать как одиночные: get, set, remove, move, copy и т.д. - все эти методы были полиморфны, т.е. могли работать как условные, так и как одиночные. Но ещё осталось несколько методово, о которых нужно обязательно сказать. Первый такой метод - это add, данный метод осуществляет вставку нового элемента в коллекцию по указанному контексту 

var db = new $.Collection([]);

// добавим новый элемент в активную коллекцию
db.add({name: "Koba"});
// теперь добавим новому элементу дополнительное свойство
db.newContext(0).add(22, "age");

Разбор полётов: первым входным параметром метод принимает новый элемент (активный контекст влияет на то, куда будет вставлен элемент), вторым входным параметром можно указать ключ элемента (если вставляем в объект, в случае если не укажем, то ключ будет сгенерирова автоматически и будет равен количеству элементов в объекте) или тип добавления данных: push или unshift (в случае если нужно вставить и название ключа и тип добавление, можно написать так например: age->unshift, по умолчанию везде используется push), последним параметром метод принимает ИД коллекции.

Метод имеет обёртки push и unshift, которые имеют 2 входных параметра: новый элемент и ИД коллекции.

Следующий метод, который мы рассмотрим будет concat - этот метод осуществляет конкатенацию коллекций: 

var db = new $.Collection([1, [1, 2], 3, 4, 5]);

// простая конкатенация
db.concat([1, 2, 3]);

// конкатенация с дополнительным контекстом
db.concat([1, 2, 3], "1");

Первым параметром методо принимает коллекцию, которая учавствует в конкатенации, вторым может принимать дополнительный контекст для коллекции с которой идёт конкатенация (плюсутеся с активным контекстом), а третим ИД коллекции. В случае конкатенации массива с массивом используется простой concat, для объектов extend, а для объекта с массивом либо extend либо concat (в зависимости от порядка).
